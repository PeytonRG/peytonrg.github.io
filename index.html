SGA Elections

November 2019 - Present

Led a team of six co-op students in adding new features to Auburn University's SGA Election website to accommodate ranked choice voting, which was voted into effect by the student body in the 2019 SGA election. Much of the work in this update encompasses database schema changes to accommodate the new ranked choice voting method in addition to the existing "standard" method, which remains in effect for non-executive positions. A SQL engine also had to be created to calculate how ranked choice votes are allocated when candidates lose successive "rounds" during vote calculation. Finally, the page where students vote for candidates and the results and demographic reports for elections needed to be updated to reflect ranked choice voting results. My personal contribution to the project consisted of a complete rewrite of the front-end and back-end for casting votes to support both old and new voting methods. My redesign targeted a "mobile-first" user experience, so I created a user interface following paradigms of hierarchical menus for selecting candidates within an election position. Principal development on the update is nearing completion, and will be followed by final rounds of quality assurance, accessibility, and user acceptance testing.

AUFSTC

July 2019 - August 2019

Led a team of four co-op students in updating the Auburn University Football Student Ticketing Center (AUFSTC) for the 2019 football season. The application provides a means for students to request free tickets/guest passes, return purchased/awarded tickets to the student ticket pool, view and dispute penalties for missing/arriving late to games, etc., and for administrators to take action on ticket requests, disputed penalties, add new tickets to the student ticket pool, generate reports on attendance data, ticket request demographics, etc., as well as customize email templates that are sent programmatically when various actions are completed or as part of scheduled tasks. My particular involvement in this update centered around updating the "Email Hub" and its accompanying backend code. The Email Hub is where administrators can edit the body of the aforementioned templates, and add variables that are replaced or "filled" when an email is sent. I redesigned the schema of the database tables where the template text and permissions are stored, and rewrote the front-end markup and back-end code to accommodate those changes. Furthermore, I rewrote the underlying logic used by the application for filling and sending each of the templates by using more memory and time efficient techniques such as StringBuilder replacements and for loops rather than string replacements and foreach loops, and ultimately refactored all template filling and sending logic into two methods rather than a specialized method for each template, as was done in the previous implementation. My contributions made the code powering these features both more efficient and maintainable.
